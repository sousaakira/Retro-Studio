================================================================================
                    🐛 QUICK DEBUG REFERENCE - MONARCO
================================================================================

1. ABRIR CONSOLE DO ELECTRON
────────────────────────────────────────────────────────────────────────────────
   Modo Desenvolvimento:
   $ npm run dev

   Produção (AppImage):
   ELECTRON_ENABLE_LOGGING=1 ./Monarco-0.1.0-x86_64.AppImage
   
   Dentro do app:
   F12  ou  Ctrl+Shift+I  →  DevTools


2. USAR FUNÇÃO log() NO MAIN PROCESS
────────────────────────────────────────────────────────────────────────────────
   Sintaxe:
   log(level, context, message, data)
   
   Exemplos:
   
   // Error (sempre mostrado)
   log('error', 'ipc:workspace', 'Workspace não encontrado', { path })
   
   // Info (desenvolvimento e produção)
   log('info', 'ipc:workspace', 'Workspace carregado', { size: 1024 })
   
   // Verbose (apenas desenvolvimento)
   log('verbose', 'fs:read', 'Lendo arquivo', { file: 'main.js' })


3. MODIFICAR NÍVEL DE DEBUG
────────────────────────────────────────────────────────────────────────────────
   No electron/main.js:
   
   DEBUG.level = 'verbose'  // Mostrar tudo (padrão em dev)
   DEBUG.level = 'info'     // Apenas info + error (padrão em prod)
   DEBUG.level = 'error'    // Apenas erros


4. LOGS NO RENDERER (Vue/JavaScript)
────────────────────────────────────────────────────────────────────────────────
   console.log('Texto:', variável)          // Info
   console.info('ℹ️ Informação')           // Info destacado
   console.warn('⚠️ Aviso')                // Aviso
   console.error('❌ Erro')                // Erro
   console.table(array)                     // Tabela
   console.group('Grupo')                   // Agrupar logs
   console.groupEnd()                       // Fim do grupo


5. DEBUGAR IPC (COMUNICAÇÃO)
────────────────────────────────────────────────────────────────────────────────
   Main Process:
   log('info', 'ipc:nome-handler', 'Descrição', { dados })
   
   Renderer (Vue):
   console.log('📤 Enviando:', nomeHandler)
   const result = await ipcRenderer.invoke(nomeHandler)
   console.log('📥 Recebido:', result)


6. BREAKPOINTS NO DEVTOOLS
────────────────────────────────────────────────────────────────────────────────
   1. DevTools → Sources
   2. Clique no número da linha
   3. Código pausa naquele ponto
   4. Inspecione no Console


7. DEBUGAR MONACO EDITOR
────────────────────────────────────────────────────────────────────────────────
   No console do DevTools:
   
   window.monaco                    // Verificar se Monaco está carregado
   window.MonacoEnvironment         // Ver configuração
   window.monaco?.editor?.getEditors?.()  // Pegar instância do editor


8. DEBUGAR IA/CHAT
────────────────────────────────────────────────────────────────────────────────
   Main Process logs (já implementado):
   - log('info', 'ipc:ai:chat', '...')
   
   Verificar endpoint (terminal):
   curl -X POST http://192.168.1.18:8000/v1/chat/completions \
   curl -X POST https://ia.auth.com.br/v1/chat/completions \
     -H "Content-Type: application/json" \
     -d '{"model":"Qwen/...","messages":[{"role":"user","content":"test"}]}'


9. DEBUGAR TERMINAL PTY
────────────────────────────────────────────────────────────────────────────────
   Main Process:
   log('info', 'ipc:terminal:create', 'Criando terminal', { shell, cwd })
   
   Renderer:
   console.log('Terminal ID:', terminalId)
   console.log('Dados:', data)


10. VISUALIZAR ARQUIVO DE LOGS
────────────────────────────────────────────────────────────────────────────────
    (Se habilitado com DEBUG.file = true)
    
    Linux/Mac:
    tail -f ~/.monarco/monarco.log
    
    Windows:
    Get-Content $env:USERPROFILE\.monarco\monarco.log -Tail 20


11. CHECKLIST RÁPIDO PARA DEBUGGING
────────────────────────────────────────────────────────────────────────────────
    ☐ Abrir DevTools (F12)

13. EXEMPLO PRÁTICO: DEBUGAR WORKSPACE SELECT
────────────────────────────────────────────────────────────────────────────────
    1. Adicionar logs no main process (já feito):
       log('info', 'ipc:workspace:select', 'Iniciando...')
       log('info', 'ipc:workspace:select', 'Workspace selecionado', {path})
       log('error', 'ipc:workspace:select', 'Erro!', {error: e.message})
    
    2. Abrir DevTools: F12
    3. Ir para Console tab
    4. Clicar em "Select Workspace"
    5. Ver os logs aparecerem em tempo real
    6. Ver dados estruturados com {path}


14. COMO ADICIONAR LOGS NOVOS
────────────────────────────────────────────────────────────────────────────────
    No electron/main.js, adicione ANTES do código que quer debugar:
    
    log('info', 'contexto:operacao', 'Descrição', { variavel: valor })
    
    Exemplo:
    ipcMain.handle('meu:handler', async () => {
      log('info', 'ipc:meu:handler', 'Iniciado')  // ← Adicione aqui
      const resultado = await fazerAlgo()
      log('info', 'ipc:meu:handler', 'Concluído', { resultado })  // ← E aqui
      return resultado
    })
